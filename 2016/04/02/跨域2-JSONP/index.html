跨域之法（二）—— JSONP

  • JavaScript

jsonp可能是很多人第一次跨域使用的一种方法，使用起来比较简单，而且也不绕。

jsonp全称 json padding,为什么？接下来你就知道了

先说说JSONP是怎么产生的

  1. 当我们请求不在当前域下的文件的时候，无论是静态页面、动态网页、web服务、WCF，只要是不同源的，就会被认定为跨域，全都是失败的

  2. 不知道你有没有发现过，Web页面上调用有src属性的文件时则,不受是否跨域的影响

  3. 于是可以判断，我们可以在远程的服务器上面把数据写在js文件中，让我们页面去调用那个js文件

  4. 然后我们就结合了json这个天然的数据装载，让客户端通过与调用脚本的方式，来调用跨域服务器上动态生成的js格式文件(一般以JSON为后缀)

  5. 服务器之所以要动态生成JSON文件，目的就在于把客户端需要的数据装入进去。

  6. 客户端在对JSON文件调用成功之后，也就获得了自己所需的数据，剩下的就是按照自己需求进行处理和展现了，也就是json+padding的时候了

  7. 为了便于客户端使用数据，逐渐形成了一种非正式传输协议，人们把它称作jsonp，该协议的一个要点就是允许用户传递一个callback参数给服务端，然后服务端返回数据时会将这个callback参数作为函数名来包裹住json数据，这样客户端就可以随意定制自己的函数来自动处理返回数据了。
